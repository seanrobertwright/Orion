  <action>
    Create src/lib/utils/stale.ts:
    ```typescript
    import { differenceInDays } from 'date-fns';

    export const STALE_THRESHOLD_DAYS = 14;

    export function isApplicationStale(
      status: string,
      lastStatusChange: Date
    ): boolean {
      // Per CONTEXT.md: Applications in 'Applied' status
      // with no activity for >14 days are stale
      if (status !== 'applied') return false;
      return differenceInDays(new Date(), lastStatusChange) > STALE_THRESHOLD_DAYS;
    }
    ```

    Create src/app/api/applications/route.ts:
    - GET: List all applications for user
      - Include job details (title, company)
      - Include stale flag
      - Support filtering by status

    Create src/app/api/applications/[id]/route.ts:
    - GET: Fetch application with job, interviews, status history
    - PUT: Update application notes, appliedAt, coverLetterId

    Create src/app/api/applications/[id]/status/route.ts (CRITICAL - per CONTEXT.md):
    - PATCH: Change application status
    - **MUST log every status change to statusHistory table**
    - Request body: { status: 'applied' | 'interviewing' | 'offered' | 'rejected' | 'saved' }
    - **Allow ANY status transition** (flexible flow per CONTEXT.md)
    - Insert into statusHistory: { applicationId, fromStatus, toStatus, changedAt: now() }
    - Update application.status and application.updatedAt
    - If changing TO 'applied' and appliedAt is null, set appliedAt = now()
  </action>
  <verify>

    Change from 'saved' directly to 'offered' - allowed (flexible flow)

  </verify>
